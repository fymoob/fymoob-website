## 1. Title tags + Meta descriptions — framework data-driven

### 1.1 Dados quantitativos (referências cientificamente validadas)

| Métrica | Valor | Fonte |
|---|---|---|
| Title length ótimo | **40-60 chars** (+8.9% CTR vs outside range) | Backlinko, 4M resultados Google |
| Word count ótimo | **6-9 palavras** | Backlinko |
| Sentiment positivo | **+4.1% CTR** vs negativo | Backlinko |
| URL matching query | **+45% CTR** | Backlinko |
| **Google reescreve descriptions** | **68-71% (desktop) / 71-87% (mobile)** | Portent, Search Engine Journal |
| Description length | **150-160 chars** (138-148 pra blog com data) | Sachs Marketing 2024 |
| Meta description rewrite case real estate | CTR 1.5% → 4.8% **(+220%)** sem mudar ranking | SEOPressor |
| Position 1 CTR drop em 2026 | **-32% YoY** (AI Overviews + SERP features) | Advanced Web Ranking Q3 2025 |
| AI Overview impact em pos 1 | Reduz CTR em **50%+** | Decoding 2026 |

**Insight crítico:** o Google reescreve **68-87%** das meta descriptions. **Não vale a pena escrever descrição "perfeita SEO"** — vale a pena escrever **descrição útil ao USUÁRIO**. Se Google reescreveu, é porque a sua não casava com a query.

### 1.2 Framework FYMOOB para reescrever as 7 páginas críticas

**Template Title (40-60 chars, 6-9 palavras):**

```
[Número OU Brackets] + [Keyword principal] + [Cidade/2026] + [Marca]
```

**Template Description (150-160 chars):**

```
[Hook com benefício/número] + [Detalhe específico] + [Call to action OU credibilidade FYMOOB]
```

### 1.4 Power words em PT-BR pra setor imobiliário (validados em SERP brasileiro)

**Conversão alta:**
- "Definitivo", "Completo", "Atualizado 2026"
- "Tabela", "Comparativo", "Real" (anti-hype)
- "Por Bairro", "Por Tipo", "Por Faixa"
- "Quanto Custa", "Como Funciona", "Vale a Pena"
- "FipeZAP", "CRECI", "Caixa" (autoridade)
- Numbers: "120 Apartamentos", "60+ Empreendimentos"

**Evitar (hype/spam):**
- "Melhor", "Incrível", "Imperdível"
- "Garanta Já", "Aproveite", "Última Chance"
- Excesso de emojis

### 1.5 Por que isso vai funcionar

1. **Títulos com brackets `[2026]` `[Tabela]`** chamam atenção visual no SERP — Backlinko showed bracketed titles get ~33% more clicks.
2. **Posição 5 com title novo + description útil** = Google **menos provável de reescrever** (matches user intent).
3. **CET, FipeZAP, CRECI** são termos que **filtram tráfego sério** (lead qualificado vs curioso).

## 2. Internal linking — topic cluster strategy

### 2.1 Lições do Zillow (33M visitas/mês, 5.2M páginas)

**Arquitetura programática Zillow:**
- 5.2M páginas indexadas (vasta maioria programáticas)
- 80% dos usuários chegam via SEO orgânico
- Apenas <1.000 páginas de blog (98% é programatic listings/locations)
- 3 templates principais: (1) Property Listings, (2) Local Service Directories, (3) Mortgage rate pages

**Princípio core:** "Crawl efficiency, internal linking, page quality optimization, and rich snippets are essential for visibility" (Commenze 2024 case study)

### 2.2 Dados sobre anchor text e diversidade

| Métrica | Valor | Fonte |
|---|---|---|
| Páginas com anchor text diverso (20+ variações) | Outrank páginas com 50 anchors idênticos | seo.ai |
| Sites com diversidade alta de anchor | Ranking médio **1.3** vs **3.5** baixa diversidade | seo.ai |
| Internal links com URLs matching anchor | **+45% CTR** | Backlinko |
| Distribuição saudável de anchor (sites top) | 30-50% branded + 20-30% partial-match + <10% exact-match | The Links Guy |
| Internal links optimal density real estate | **2-5 links/página** OU 1 a cada 300 palavras | ScaleAcres |
| Site depth máxima recomendada | **3 cliques** da homepage | Topic clustering best practice |

### 2.3 Estrutura hub-and-spoke proposta para FYMOOB

```
                          HOMEPAGE (hub principal)
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
       /apartamentos-    /casas-curitiba  /lancamentos
        curitiba          /sobrados-...    /empreendimentos
              │               │               │
              ▼               ▼               ▼
     /imoveis/[bairro]  /imoveis/[bairro]/  /empreendimento/
      (36 hubs)          [tipo] (110 spokes) [slug] (60 spokes)
              │               │               │
              ▼               ▼               ▼
      /imovel/[slug] (250+ leaf pages)
              ▲
              │ (LINKAGEM CRUZADA)
   /comprar-apartamento-curitiba (pillar) ─┐
   /comprar-imovel-curitiba (pillar)       │ Top of authority
   /morar-em-curitiba (pillar)             │
   /alugar-curitiba (pillar)               │
   /blog/[slug] (15 spokes editoriais)     │
```

### 2.4 Ações específicas para FYMOOB (com base nos dados GSC)

#### Empreendimentos descobertos em pos 20-90 — empurrar pra top 10

**Diagnóstico:** estes empreendimentos estão chegando ao Google mas com **autoridade interna fraca** (poucos links chegam neles). Reforçar internal linking eleva.

**Lista detectada no GSC:**
- /empreendimento/orfeu (pos 20.5)
- /empreendimento/zait (pos 20)
- /empreendimento/myta (pos 14)
- /empreendimento/mova-wf (pos 24)
- /empreendimento/lemme (pos 35)
- /empreendimento/lange (pos 39)
- /empreendimento/portao-prime-residence (pos 53.67)
- /empreendimento/le-sense (pos 82)

**Ações concretas:**

**A) Footer global rotativo (mais forte mudança)**
Adicionar 8 empreendimentos rotacionados no footer global do site:
```html
<div className="footer-empreendimentos">
  <h3>Empreendimentos em Curitiba</h3>
  {empreendimentosTop8.map(e => (
    <Link href={`/empreendimento/${e.slug}`} key={e.slug}>
      {e.nome} — {e.bairro}
    </Link>
  ))}
</div>
```
Rotação por hash (estável por dia, mas variando ao longo da semana). **Impact:** cada empreendimento ganha ~250 internal links instantaneamente (uma por página do site).

**B) "Empreendimentos parecidos" em cada `/empreendimento/[slug]`**
Component que linka 3-4 empreendimentos do mesmo bairro / mesma faixa de preço:
```jsx
<RelatedEmpreendimentos empreendimento={emp} count={4} criteria={["bairro", "preco-similar"]} />
```

**C) Bairro pages linkam empreendimentos do bairro**
Em `/imoveis/portao` adicionar seção "Empreendimentos em Portão" listando todos com 2+ unidades.

**D) Blog posts mencionam empreendimentos relevantes**
Quando blog post fala sobre Mossunguê → link pra /empreendimento/reserva-barigui.
Quando fala sobre Portão → link pra /empreendimento/le-monde, /empreendimento/easy.

**E) Anchor text diversificado**

Para cada empreendimento, usar **5-8 anchors diferentes** distribuídos pelo site:
- "Reserva Barigui" (branded) — 30-40%
- "Reserva Barigui em Mossunguê" (branded + local)
- "Apartamento Reserva Barigui" (branded + tipo)
- "Empreendimento Reserva Barigui Curitiba" (long tail)
- "lançamento Avantti em Mossunguê" (descriptive, sem brand)
- "veja empreendimento" (CTA, raro)
- (URL nua: 5%)

### 2.5 Estimativa quantitativa

Com 250 páginas linkando empreendimentos via footer (B) + 5-10 contextual links (C, D) + 4 similar (E), cada empreendimento sai de ~5 internal links pra **~280 internal links**.

**Esperado em 60-90 dias:**
- Empreendimentos pos 20-90 → pos 10-15
- Empreendimentos com volume alto (Reserva Barigui, Le Monde) → top 5

## 3. Schema markup — escalar Apartment

### 3.1 CTR boost validado por estudos

| Fonte | CTR boost com schema |
|---|---|
| Search Engine Journal (2024) | **35-45%** com rich snippets |
| Schema App (study) | **+18%** |
| Google Search Central (case studies) | **+25% CTR + 35% mais visitas** |
| Industry average (real estate) | **20-30%** |
| **FYMOOB validado no GSC** | **25%** (Snippets do produto: 2 cliques de 8 imp) ✅ |

### 3.2 Schemas reais que o Google mostra como rich snippet em 2026

**Aceitos pelo Google para imóveis:**
- ✅ `RealEstateListing` (cobertura geral)
- ✅ `Apartment` (subtipo específico)
- ✅ `House` / `SingleFamilyResidence`
- ✅ `Place` (para terrenos)
- ⚠️ `AggregateOffer` (para empreendimentos com 50+ unidades)
- ❌ `AggregateRating` em listings individuais — **Google rejeita** (review schema só em business/products)

**O que aparece visualmente no SERP:**
- Preço (via `offers.price`)
- Imagem (via `image`)
- Endereço (via `address`)
- Categoria (apartamento/casa)
- Snippet de produto (✅ FYMOOB já viu 25% CTR)

**O que NÃO aparece:**
- Estrelas (só em Organization/LocalBusiness, não em listings)
- Number of rooms direto (vai no description)

### 3.3 Estado atual FYMOOB vs o que falta

#### O que já temos (Fase 19.P0)
- ✅ `Apartment/House/Place` schema rico em ItemList em **11 páginas**
- ✅ `RealEstateListing` em /imovel/[slug] (250+ pages)
- ✅ `BreadcrumbList` em todas listings
- ✅ `FAQPage` em ~15 páginas
- ✅ `Organization` + `LocalBusiness` em layout raiz
- ✅ `Dataset` em /imoveis/[bairro]

#### Onde ainda falta (gaps detectáveis)

**A) Empreendimentos individuais — `/empreendimento/[slug]` precisa schema mais rico**

Hoje provavelmente só tem `RealEstateListing` básico. Adicionar:
- `AggregateOffer` (price range desde menor unidade até maior)
- `numberOfBedroomOptions` (todas variantes 1qto, 2qto, 3qto)
- `floorSize` range (m² menor → m² maior)
- Embed cada `Apartment` individual da lista

**B) Bairro pages — adicionar `Place` schema**

`/imoveis/[bairro]` é uma listing geográfica. Schema Place rico:
```json
{
  "@type": "Place",
  "name": "Portão, Curitiba",
  "containedInPlace": { "@type": "City", "name": "Curitiba" },
  "geo": { "@type": "GeoCoordinates", "latitude": -25.45, "longitude": -49.28 },
  "amenityFeature": [...] // shopping, parques, escolas
}
```

**C) Blog posts — `Article` schema com `headline`, `datePublished`, `author`, `image`**

Verificar se já tem (provavelmente sim via BlogPosting), mas adicionar:
- `mentions` (array de Place pros bairros mencionados)
- `about` (RealEstateListing relevante)

### 3.4 Plano de escalada

**Quick wins (3-4h):**
1. Adicionar `AggregateOffer` em cada `/empreendimento/[slug]` (rich snippet com faixa de preço "R$ 350 mil — R$ 1,2 mi")
2. Verificar/adicionar `mentions` + `about` nos blog posts pra reforçar conexão entidade-bairros-empreendimentos
3. Garantir `image` URL **absoluta** em todos os schemas (não relativa)

**Médio prazo (P1):**
4. `Place` schema rico nos 36 bairros (com `amenityFeature` listando POIs)
5. `HousingDevelopment` ou `ApartmentComplex` quando schema.org adicionar oficialmente (atualmente em alpha)
