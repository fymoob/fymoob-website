# Hipótese H-20260417-003 — React Compiler stable

## Believe That
Habilitar `experimental.reactCompiler: true` em [next.config.ts](../../../next.config.ts) (Next 16.1.6 + React 19.2.3 estável).

## Will Result In

- **TBT mediana cai 10-30%** na rota `/busca` (reduz work de reconciliation por memoização automática)
- **INP melhora em interações** (abrir Sheet, trocar filtro) — menos re-renders durante user interactions
- **LCP possivelmente melhora marginal** (~50-150ms) se hidratação também ficar mais leve
- Bundle raw provavelmente neutro ou +3-8 KB (compiler adiciona pequenas helpers)

**Mecanismo:** React Compiler (ex-Forget) memoiza automaticamente componentes e callbacks em compile-time, eliminando re-renders desnecessários sem precisar `useMemo`/`useCallback` manual. Meta relata em seu codebase: **20-40% menos re-renders**, +12% LCP, 2.5x melhoria de interações ([InfoQ, dez/2025](https://www.infoq.com/news/2025/12/react-compiler-meta/), [digitalapplied.com](https://www.digitalapplied.com/blog/react-compiler-stable-nextjs-16-automatic-memoization)).

**Componentes que mais podem se beneficiar no /busca:**
- `PropertyCard` (renderizado 24×, recebe props complexas)
- `usePropertyCard` hook (lógica com state derivado)
- `SearchBar` + `useSearchBarController` (pendingFilters trigger re-renders em toda input)
- `PhotoCarousel` (nossa nova versão)

## Success Criterion

Em `/busca` mobile, 5 runs pós-deploy:
- **TBT mediana: < 600ms** (baseline 645ms; meta é -7%+)
- **CoV < 10%** (conforme STATISTICAL-RIGOR.md)
- **Bundle /busca total: ≤ 770 KB** (baseline 762 KB; aceito +8 KB margem)
- **LCP mediana: ≤ 4.90s** (baseline 4.84s; aceito +1% margem)
- Build passa sem warnings novos do compiler

## Kill Criterion

Reverter `reactCompiler: true` → `false` se:
- Build quebrar com erros de compiler (não corrigíveis em 20min)
- TBT mediana piorar >50ms (regressão real de performance)
- Bundle crescer > 30 KB (compiler gerou código demais)
- Qualquer teste visual manual detectar regressão de comportamento
- CLS piorar significativamente (>0.01 na mediana)
- Algum console error em runtime
