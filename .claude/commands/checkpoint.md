# Checkpoint — Salvar Progresso da Sessão

Antes de encerrar esta sessão, execute o seguinte:

1. **Leia `docs/sprint-plan.md`** e identifique a fase/sprint atual
2. **Marque como concluídas** `[x]` todas as tarefas que foram completadas nesta sessão
3. **Crie/atualize `docs/session-log.md`** adicionando uma entrada com:
   - Data e hora
   - Fase/Sprint trabalhada
   - Tarefas concluídas (lista)
   - Arquivos criados ou modificados (lista)
   - Decisões de arquitetura tomadas (se houver)
   - Problemas encontrados (se houver)
   - Próxima tarefa a fazer quando retomar
4. **Atualize o CLAUDE.md** se alguma convenção ou padrão novo foi definido durante a sessão
5. **Faça um git commit** com a mensagem: `checkpoint: [fase] - [resumo breve]`

Formato da entrada no session-log.md:
```markdown
## Sessão [DATA]
**Fase:** [N] — [Nome]
**Tarefas concluídas:**
- [tarefa 1]
- [tarefa 2]

**Arquivos modificados:**
- [path/arquivo1]
- [path/arquivo2]

**Decisões:**
- [decisão tomada e motivo]

**Próxima tarefa:**
- [o que fazer quando retomar]
```

Após o checkpoint, confirme o que foi salvo.
